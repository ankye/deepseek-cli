# Glossary / 术语表

| Term / 术语 | Meaning / 含义 |
| --- | --- |
| Host | CLI, VSCode, server, SDK, native, or browser surface that renders events and collects input. / 渲染事件并收集输入的 CLI、VSCode、server、SDK、native 或 browser 表面。 |
| Runtime kernel | Headless execution owner that builds envelopes, calls policy, schedules work, and emits events. / 构建 envelope、调用 policy、调度工作并发出事件的 headless 执行 owner。 |
| Execution envelope | Required metadata wrapper for executable work. / 可执行工作的必需 metadata 包装。 |
| Capability | Discoverable or executable platform contribution such as tool, command, skill, hook, MCP, plugin, workflow, subagent. / 工具、命令、skill、hook、MCP、plugin、workflow、subagent 等可发现或可执行平台贡献。 |
| Workflow | Task graph describing what work exists and when it is complete. / 描述有哪些工作以及何时完成的任务图。 |
| Scheduler | System deciding when governed work runs. / 决定受治理工作何时运行的系统。 |
| Resource lock | Metadata used to serialize conflicting tasks. / 用于串行化冲突任务的 metadata。 |
| Policy | Decision layer for allow, ask, deny, rewrite, require-sandbox. / allow、ask、deny、rewrite、require-sandbox 决策层。 |
| Sandbox | Runtime/platform restrictions required for side-effecting work. / 带副作用工作所需的 runtime/platform 限制。 |
| Platform matrix | Descriptor of OS, shell, process, filesystem, network, native, secure storage capabilities. / OS、shell、process、filesystem、network、native、secure storage 能力描述。 |
| Redaction | Transforming sensitive content into safe evidence. / 将敏感内容转换为安全证据。 |
| Golden replay | Deterministic trace replay test. / 确定性 trace replay 测试。 |
| OpenSpec | Formal change/spec workflow used for requirements and acceptance. / 用于需求和验收的正式 change/spec 工作流。 |
