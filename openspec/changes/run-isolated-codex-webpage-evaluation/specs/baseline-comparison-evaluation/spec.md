## ADDED Requirements

### Requirement: Baseline comparison evaluates DeepSeek Codex and Claude together / Baseline 对比同时评估 DeepSeek、Codex 与 Claude
The system SHALL support a comparison evaluation that includes DeepSeek CLI, Codex, Claude Code, and future baselines under the same selected task definitions.

系统必须支持 comparison evaluation，在相同 selected task definitions 下同时包含 DeepSeek CLI、Codex、Claude Code 与未来 baseline。

#### Scenario: Multi-baseline dry-run comparison / 多 Baseline Dry-run 对比
- **WHEN** `deepseek diagnostics evaluate --full --compare-baseline deepseek-cli --compare-baseline codex --compare-baseline claude-code --dry-run --output json` runs
- **THEN** the evaluation summary includes all requested baselines, selected task runs for each baseline, baseline aggregates, and gap finding placeholders without executing external CLIs
- **中文** 当运行 `deepseek diagnostics evaluate --full --compare-baseline deepseek-cli --compare-baseline codex --compare-baseline claude-code --dry-run --output json` 时，evaluation summary 必须包含所有 requested baselines、每个 baseline 的 selected task runs、baseline aggregates 与 gap finding placeholders，且不得执行 external CLIs。

### Requirement: Baseline comparison records product quality metrics / Baseline 对比记录产品质量指标
The system SHALL record task outcome metrics and product operating metrics from instrumentation events, process results, checker output, and artifact scans, including first-run success, command success rate, check pass rate, correction count, command failure count, elapsed time, generated artifact structure, and bounded output previews.

系统必须从 instrumentation events、process results、checker output 与 artifact scans 中记录 task outcome metrics 与 product operating metrics，包括首轮成功、命令成功率、check pass rate、纠错次数、命令失败次数、耗时、生成产物结构与有界输出预览。

#### Scenario: Executed run records gap-relevant metrics / 已执行 Run 记录差距相关指标
- **WHEN** a baseline executes a webpage-generation task
- **THEN** the task run record includes outcome, check results, instrumentation events, elapsed time, first-run success, command run count, command success count, command success rate, generated file counts, generated byte totals, code structure score, correction count, command failure count, diagnostics, evidence paths, and redaction metadata
- **中文** 当某个 baseline 执行 webpage-generation task 时，task run record 必须包含 outcome、check results、instrumentation events、elapsed time、first-run success、command run count、command success count、command success rate、generated file counts、generated byte totals、code structure score、correction count、command failure count、diagnostics、evidence paths 与 redaction metadata。

#### Scenario: Instrumentation events back derived statistics / 插桩事件支撑派生统计
- **WHEN** a baseline task run is executed
- **THEN** the run emits machine-readable lifecycle events for run start, isolated workspace creation, prompt write/send, baseline command start/finish, checker start/finish, artifact scan start/finish, evidence write, and run finish
- **中文** 当 baseline task run 被执行时，run 必须输出机器可读的生命周期事件，覆盖 run start、isolated workspace creation、prompt write/send、baseline command start/finish、checker start/finish、artifact scan start/finish、evidence write 与 run finish。

### Requirement: Baseline comparison derives aggregate gaps / Baseline 对比推导聚合差距
The system SHALL derive per-baseline aggregates and gap findings from task-run records without replacing the underlying evidence.

系统必须从 task-run records 推导每个 baseline 的 aggregates 与 gap findings，但不得替代底层证据。

#### Scenario: Gap findings cite metric source / 差距发现引用指标来源
- **WHEN** comparison summary is rendered
- **THEN** it includes baseline aggregate success rates, command success rates, check pass rates, average structure scores, correction rates, and gap findings that name the metric and compared baselines
- **中文** 当 comparison summary 被渲染时，它必须包含 baseline aggregate success rates、command success rates、check pass rates、average structure scores、correction rates，以及命名 metric 与 compared baselines 的 gap findings。

### Requirement: External baseline execution remains explicit and isolated / External Baseline 执行保持显式且隔离
The system SHALL execute Codex or Claude only after explicit maintainer opt-in, and every execution SHALL occur in an isolated workspace outside the repository root.

系统只有在维护者显式 opt-in 后才能执行 Codex 或 Claude，且每次执行都必须发生在仓库根目录之外的隔离 workspace 中。

#### Scenario: External baseline cannot execute by accident / External Baseline 不会意外执行
- **WHEN** a comparison includes `codex` or `claude-code` without `--allow-external-baseline` or without `--execute-task`
- **THEN** those baselines are probed or deferred according to configuration, but task prompts are not sent and the repository workspace is not mutated
- **中文** 当 comparison 包含 `codex` 或 `claude-code` 但没有 `--allow-external-baseline` 或没有 `--execute-task` 时，这些 baseline 只能按配置被 probe 或 deferred，不得发送 task prompt，也不得修改仓库 workspace。

#### Scenario: Executed external task uses isolated workspace / 已执行 External Task 使用隔离 Workspace
- **WHEN** an external baseline executes `eval.webpage.generation`
- **THEN** the process cwd and generated artifact path are under an isolated evaluation workspace, and check commands run against that workspace's generated artifacts
- **中文** 当 external baseline 执行 `eval.webpage.generation` 时，process cwd 与 generated artifact path 必须位于隔离 evaluation workspace 下，check commands 必须针对该 workspace 的 generated artifacts 运行。
