## ADDED Requirements

### Requirement: Pipeline Block Projection / 管道块投影

The context engine SHALL project eligible context graph nodes into immutable context blocks assigned to canonical pipeline layers while preserving existing budget, redaction, cache, ordering, and exclusion rules.

Context engine 必须将 eligible context graph nodes 投影为分配到规范 pipeline layers 的不可变 context blocks，同时保留现有 budget、redaction、cache、ordering 与 exclusion rules。

#### Scenario: Context nodes become blocks / Context Nodes 转为 Blocks

- **WHEN** runtime requests model context for a turn
- **THEN** context projection returns selected nodes and a pipeline manifest whose blocks contain hashes, layer ids, token estimates, dependency fingerprints, provenance, and redaction metadata
- **中文** 当 runtime 请求某 turn 的 model context 时，context projection 必须返回 selected nodes 和 pipeline manifest，manifest 中 blocks 包含 hashes、layer ids、token estimates、dependency fingerprints、provenance 与 redaction metadata。

#### Scenario: Existing projection fallback remains available / 保留现有投影回退

- **WHEN** a caller does not request pipeline output
- **THEN** context-engine preserves the existing `ContextProjectionResult` behavior while still being able to derive a compatible manifest in tests or diagnostics
- **中文** 当 caller 未请求 pipeline output 时，context-engine 必须保留现有 `ContextProjectionResult` 行为，同时仍能在测试或 diagnostics 中派生兼容 manifest。

### Requirement: Stable Layer Assignment / 稳定层分配

The context engine SHALL assign nodes to pipeline layers by lifecycle, source, and volatility rather than by one-turn priority alone.

Context engine 必须根据 lifecycle、source 与 volatility 将 nodes 分配到 pipeline layers，而不是只按单回合 priority。

#### Scenario: Project facts stay in project layer / 项目事实保留在 Project 层

- **WHEN** repository rules, package metadata, package map, or governed project memory are eligible
- **THEN** projection assigns them to the `project` layer unless policy or freshness metadata requires exclusion
- **中文** 当 repository rules、package metadata、package map 或受治理 project memory eligible 时，projection 必须将其分配到 `project` layer，除非 policy 或 freshness metadata 要求排除。

#### Scenario: Volatile evidence stays in current turn / 易变证据保留在当前回合

- **WHEN** evidence is generated by the current user input, active selection, or full raw tool output
- **THEN** projection assigns it to `current-turn` unless it has been summarized into a bounded session evidence block
- **中文** 当 evidence 来自当前用户输入、活动选择或完整 raw tool output 时，projection 必须将其分配到 `current-turn`，除非它已被摘要为有界 session evidence block。

### Requirement: Prefix Stability Diagnostics / 前缀稳定性诊断

The context engine SHALL report prefix stability metadata without exposing raw block content.

Context engine 必须报告 prefix stability metadata，且不暴露 raw block content。

#### Scenario: Layer hash changes are reported / 层 Hash 变化被报告

- **WHEN** the generated pipeline manifest changes relative to a previous manifest for the same session
- **THEN** diagnostics identify changed layer hashes, changed block ids, changed dependency fingerprints, and estimated affected tokens
- **中文** 当生成的 pipeline manifest 相对同一 session 的前一个 manifest 发生变化时，diagnostics 必须识别变化的 layer hashes、block ids、dependency fingerprints 与估算受影响 tokens。
